1. Tratamento de Exceções

O que são exceções?

Exceções são eventos que ocorrem durante a execução de um programa e que interrompem o fluxo normal de instruções.

Por que usar exceções?

  • Separar o código de tratamento de erros do código principal
  • Propagar erros para cima na pilha de chamadas
  • Agrupar e diferenciar tipos de erros
  • Garantir que recursos sejam liberados corretamente

Hierarquia de Exceções em Java

Checked Exceptions

  • Verificadas em tempo de compilação
  • Obrigam o programador a tratá-las
  • Exemplo: IOException, SQLException

Unchecked Exceptions

  • Não verificadas em tempo de compilação
  • Derivadas de RuntimeException
  • Exemplo: NullPointerException, ArrayIndexOutOfBoundsException

Blocos Try-Catch

Estrutura básica para tratamento de exceções:

try { // Código que pode gerar exceção int resultado = 10 / 0; // ArithmeticException } catch (ArithmeticException e) { // Tratamento para ArithmeticException System.out.println("Erro: Divisão por zero!"); e.printStackTrace(); } catch (Exception e) { // Tratamento genérico para outras exceções System.out.println("Ocorreu um erro: " + e.getMessage()); } finally { // Código que sempre será executado System.out.println("Finalizando operação..."); }
Nota: O bloco finally é opcional, mas muito útil para garantir que recursos sejam liberados, independentemente de ocorrer ou não uma exceção.

Recursos Avançados de Try-Catch

Multi-catch (Java 7+)

try { // Código que pode gerar diferentes exceções } catch (IOException | SQLException e) { // Tratamento comum para ambas as exceções System.out.println("Erro de I/O ou banco de dados: " + e.getMessage()); }

Try-with-resources (Java 7+)

// Automaticamente fecha recursos que implementam AutoCloseable try (FileInputStream fis = new FileInputStream("arquivo.txt"); BufferedReader br = new BufferedReader(new InputStreamReader(fis))) { String linha; while ((linha = br.readLine()) != null) { System.out.println(linha); } } catch (IOException e) { System.err.println("Erro ao ler arquivo: " + e.getMessage()); } // Não é necessário bloco finally para fechar recursos!

Criando Exceções Personalizadas

É possível criar suas próprias classes de exceção para representar erros específicos do seu domínio de aplicação:

// Criando uma exceção checada public class SaldoInsuficienteException extends Exception { private double saldoAtual; private double valorSaque; public SaldoInsuficienteException( double saldoAtual, double valorSaque) { super("Saldo insuficiente: " + saldoAtual + ", Saque: " + valorSaque); this.saldoAtual = saldoAtual; this.valorSaque = valorSaque; } public double getSaldoAtual() { return saldoAtual; } public double getValorSaque() { return valorSaque; } }

Usando a exceção personalizada:

public class ContaBancaria { private double saldo; public void sacar(double valor) throws SaldoInsuficienteException { if (valor > saldo) { throw new SaldoInsuficienteException(saldo, valor); } saldo -= valor; } // Exemplo de uso public static void main(String[] args) { ContaBancaria conta = new ContaBancaria(); try { conta.sacar(100.0); } catch (SaldoInsuficienteException e) { System.out.println(e.getMessage()); // Acesso aos dados específicos System.out.println("Saldo: " + e.getSaldoAtual()); } } }

2. Gerenciamento de Arquivos

Classes principais para manipulação de arquivos

java.io (Clássico)

  • File: representação de arquivos e diretórios
  • FileInputStream/FileOutputStream: operações com bytes
  • FileReader/FileWriter: operações com caracteres
  • BufferedReader/BufferedWriter: operações bufferizadas

java.nio (Java 7+)

  • Path: representação moderna de arquivos
  • Files: classe utilitária com métodos estáticos
  • Charset: representação de conjuntos de caracteres
  • APIs não bloqueantes para operações de I/O
Observação: Operações com arquivos geralmente exigem tratamento de exceções, pois são frequentes fontes de erros em aplicações.

Leitura de Arquivos

Com java.io

try (FileReader fr = new FileReader("dados.txt"); BufferedReader br = new BufferedReader(fr)) { String linha; while ((linha = br.readLine()) != null) { System.out.println(linha); } } catch (IOException e) { System.err.println("Erro ao ler arquivo: " + e.getMessage()); }

Com java.nio (Java 7+)

try { Path path = Paths.get("dados.txt"); // Lendo todas as linhas de uma vez List linhas = Files.readAllLines( path, StandardCharsets.UTF_8); for (String linha : linhas) { System.out.println(linha); } // Ou usando streams (Java 8+) Files.lines(path, StandardCharsets.UTF_8) .forEach(System.out::println); } catch (IOException e) { System.err.println("Erro: " + e.getMessage()); }

Escrita em Arquivos

Com java.io

try (FileWriter fw = new FileWriter("saida.txt"); BufferedWriter bw = new BufferedWriter(fw)) { bw.write("Primeira linha de texto"); bw.newLine(); // Quebra de linha bw.write("Segunda linha de texto"); bw.flush(); // Força a escrita dos dados } catch (IOException e) { System.err.println("Erro ao escrever: " + e.getMessage()); }

Com java.nio (Java 7+)

try { Path path = Paths.get("saida.txt"); // Escrevendo uma lista de strings List linhas = Arrays.asList( "Primeira linha de texto", "Segunda linha de texto" ); Files.write(path, linhas, StandardCharsets.UTF_8, StandardOpenOption.CREATE, StandardOpenOption.WRITE); } catch (IOException e) { System.err.println("Erro: " + e.getMessage()); }
Atenção: Por padrão, o método write sobrescreve arquivos existentes. Para preservar conteúdo ou acrescentar texto, use o FileWriter("arquivo.txt", true) ou StandardOpenOption.APPEND.

Serialização de Objetos

Serialização é o processo de converter objetos em um formato que pode ser salvo em disco ou transmitido pela rede.

Criando uma classe serializável

import java.io.Serializable; public class Pessoa implements Serializable { // Recomendado para controle de versão private static final long serialVersionUID = 1L; private String nome; private int idade; private transient String senhaTemporaria; // Construtores, getters e setters @Override public String toString() { return "Pessoa [nome=" + nome + ", idade=" + idade + "]"; } }

Salvando e carregando objetos

// Salvando objeto try (FileOutputStream fos = new FileOutputStream("pessoa.ser"); ObjectOutputStream oos = new ObjectOutputStream(fos)) { Pessoa p = new Pessoa("João", 30); oos.writeObject(p); } catch (IOException e) { e.printStackTrace(); } // Carregando objeto try (FileInputStream fis = new FileInputStream("pessoa.ser"); ObjectInputStream ois = new ObjectInputStream(fis)) { Pessoa p = (Pessoa) ois.readObject(); System.out.println(p); } catch (IOException | ClassNotFoundException e) { e.printStackTrace(); }
Nota: O modificador transient indica que um atributo não deve ser serializado.

3. Interface Gráfica com Swing

O que é Swing?

  • Biblioteca Java para criação de interfaces gráficas
  • Parte do JFC (Java Foundation Classes)
  • Implementa o padrão MVC (Model-View-Controller)
  • Oferece componentes GUI leves e portáveis
  • Altamente customizável com Look and Feel

Componentes principais

  • Contêineres: JFrame, JPanel, JDialog
  • Controles: JButton, JTextField, JComboBox
  • Menus: JMenuBar, JMenu, JMenuItem
  • Layout Managers para organização dos componentes
import javax.swing.*; public class MinhaPrimeiraJanela { public static void main(String[] args) { // Para segurança de thread em Swing SwingUtilities.invokeLater(() -> { // Cria uma janela JFrame janela = new JFrame("Olá Swing"); // Define operação de fechamento janela.setDefaultCloseOperation( JFrame.EXIT_ON_CLOSE); // Define tamanho janela.setSize(400, 300); // Centraliza na tela janela.setLocationRelativeTo(null); // Torna visível janela.setVisible(true); }); } }

Layout Managers

Gerenciadores de layout controlam o posicionamento e o dimensionamento dos componentes.

Principais Layout Managers

  • BorderLayout: divide em 5 regiões (North, South, East, West, Center)
  • FlowLayout: organiza componentes em linha, da esquerda para a direita
  • GridLayout: organiza em grade com linhas e colunas de mesmo tamanho
  • BoxLayout: organiza componentes em linha horizontal ou vertical
  • GridBagLayout: o mais flexível, mas também o mais complexo
JFrame janela = new JFrame("Exemplo de Layout"); janela.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); janela.setSize(400, 300); // Painel com BorderLayout (padrão para JFrame) JPanel painelPrincipal = new JPanel(); painelPrincipal.setLayout(new BorderLayout(5, 5)); // Componentes em diferentes regiões JButton btnNorte = new JButton("Norte"); JButton btnSul = new JButton("Sul"); JTextArea txtCentro = new JTextArea("Centro"); painelPrincipal.add(btnNorte, BorderLayout.NORTH); painelPrincipal.add(btnSul, BorderLayout.SOUTH); painelPrincipal.add(txtCentro, BorderLayout.CENTER); janela.setContentPane(painelPrincipal); janela.setVisible(true);

Componentes Básicos do Swing

Componentes de Entrada

  • JTextField: campo de texto de uma linha
  • JPasswordField: campo para senhas
  • JTextArea: área de texto multilinhas
  • JComboBox: lista suspensa de opções
  • JCheckBox: caixa de seleção
  • JRadioButton: botão de opção (em grupo)
  • JSlider: controle deslizante

Componentes de Ação e Visualização

  • JButton: botão clicável
  • JLabel: texto e/ou imagem não editável
  • JList: lista de itens selecionáveis
  • JTable: dados em formato tabular
  • JTree: dados em formato hierárquico
  • JProgressBar: barra de progresso
  • JSplitPane: divisão ajustável de painéis

Tratamento de Eventos

Interfaces gráficas são baseadas em eventos: responder a ações do usuário como cliques, digitação, etc.

Principais tipos de listeners

  • ActionListener: para botões, menus, etc.
  • MouseListener: para eventos de mouse
  • KeyListener: para eventos de teclado
  • WindowListener: para eventos da janela
  • FocusListener: para ganho/perda de foco
// Usando classes anônimas (tradicional) JButton btnSalvar = new JButton("Salvar"); btnSalvar.addActionListener(new ActionListener() { @Override public void actionPerformed(ActionEvent e) { System.out.println("Botão clicado!"); // Lógica de salvamento } }); // Usando lambda (Java 8+) JButton btnCancelar = new JButton("Cancelar"); btnCancelar.addActionListener(e -> { System.out.println("Cancelado!"); // Lógica de cancelamento });
Dica: Para interfaces de listener com muitos métodos, considere usar classes adaptadoras como MouseAdapter, WindowAdapter, etc., que fornecem implementações vazias de todos os métodos.

Caixas de Diálogo

O Swing oferece caixas de diálogo pré-definidas para interações comuns com o usuário:

JOptionPane

  • showMessageDialog: exibe uma mensagem
  • showConfirmDialog: pede confirmação (Sim/Não/Cancelar)
  • showInputDialog: solicita entrada de texto
  • showOptionDialog: oferece opções personalizadas

Diálogos de Arquivo

  • JFileChooser: para seleção de arquivos
  • FileNameExtensionFilter: filtro de tipos de arquivo
// Mensagem simples JOptionPane.showMessageDialog( null, "Operação realizada com sucesso!", "Informação", JOptionPane.INFORMATION_MESSAGE ); // Confirmação int opcao = JOptionPane.showConfirmDialog( null, "Deseja salvar as alterações?", "Confirmação", JOptionPane.YES_NO_CANCEL_OPTION ); // Seletor de arquivos JFileChooser fileChooser = new JFileChooser(); fileChooser.setDialogTitle("Selecione um arquivo"); // Filtro para arquivos de texto FileNameExtensionFilter filtro = new FileNameExtensionFilter( "Arquivos de texto (*.txt)", "txt"); fileChooser.addChoosableFileFilter(filtro); int resultado = fileChooser.showOpenDialog(null); if (resultado == JFileChooser.APPROVE_OPTION) { File arquivoSelecionado = fileChooser.getSelectedFile(); // Processar o arquivo }

Projeto Integrado: Editor de Notas

Vamos criar um exemplo que integra tratamento de exceções, manipulação de arquivos e interface Swing.

import javax.swing.*; import java.awt.*; import java.awt.event.*; import java.io.*; public class EditorDeNotas extends JFrame { private JTextArea areaTexto; private JFileChooser seletorArquivo; private File arquivoAtual; public EditorDeNotas() { // Configuração da janela super("Editor de Notas"); setSize(600, 400); setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); setLocationRelativeTo(null); // Componentes areaTexto = new JTextArea(); areaTexto.setFont(new Font("Monospaced", Font.PLAIN, 14)); JScrollPane scrollPane = new JScrollPane(areaTexto); seletorArquivo = new JFileChooser(); // Menu JMenuBar barraMenu = new JMenuBar(); JMenu menuArquivo = new JMenu("Arquivo"); JMenuItem itemNovo = new JMenuItem("Novo"); JMenuItem itemAbrir = new JMenuItem("Abrir"); JMenuItem itemSalvar = new JMenuItem("Salvar"); JMenuItem itemSair = new JMenuItem("Sair"); // Adiciona os itens ao menu menuArquivo.add(itemNovo); menuArquivo.add(itemAbrir); menuArquivo.add(itemSalvar); menuArquivo.addSeparator(); menuArquivo.add(itemSair); barraMenu.add(menuArquivo); // Listeners itemNovo.addActionListener(e -> novoArquivo()); itemAbrir.addActionListener(e -> abrirArquivo()); itemSalvar.addActionListener(e -> salvarArquivo()); itemSair.addActionListener(e -> System.exit(0)); // Layout setJMenuBar(barraMenu); getContentPane().add(scrollPane, BorderLayout.CENTER); } private void novoArquivo() { areaTexto.setText(""); arquivoAtual = null; setTitle("Editor de Notas"); } private void abrirArquivo() { try { int retorno = seletorArquivo.showOpenDialog(this); if (retorno == JFileChooser.APPROVE_OPTION) { arquivoAtual = seletorArquivo.getSelectedFile(); lerArquivo(arquivoAtual); setTitle("Editor de Notas - " + arquivoAtual.getName()); } } catch (Exception e) { JOptionPane.showMessageDialog(this, "Erro ao abrir arquivo: " + e.getMessage(), "Erro", JOptionPane.ERROR_MESSAGE); } } private void lerArquivo(File arquivo) { try (BufferedReader reader = new BufferedReader( new FileReader(arquivo))) { StringBuilder conteudo = new StringBuilder(); String linha; while ((linha = reader.readLine()) != null) { conteudo.append(linha).append("\n"); } areaTexto.setText(conteudo.toString()); } catch (IOException e) { throw new RuntimeException("Não foi possível ler o arquivo", e); } } private void salvarArquivo() { try { if (arquivoAtual == null) { int retorno = seletorArquivo.showSaveDialog(this); if (retorno == JFileChooser.APPROVE_OPTION) { arquivoAtual = seletorArquivo.getSelectedFile(); setTitle("Editor de Notas - " + arquivoAtual.getName()); } else { return; // Usuário cancelou } } escreverArquivo(arquivoAtual); JOptionPane.showMessageDialog(this, "Arquivo salvo com sucesso!", "Informação", JOptionPane.INFORMATION_MESSAGE); } catch (Exception e) { JOptionPane.showMessageDialog(this, "Erro ao salvar arquivo: " + e.getMessage(), "Erro", JOptionPane.ERROR_MESSAGE); } } private void escreverArquivo(File arquivo) { try (BufferedWriter writer = new BufferedWriter( new FileWriter(arquivo))) { writer.write(areaTexto.getText()); } catch (IOException e) { throw new RuntimeException("Não foi possível salvar o arquivo", e); } } public static void main(String[] args) { SwingUtilities.invokeLater(() -> { new EditorDeNotas().setVisible(true); }); } }

Boas Práticas

Tratamento de Exceções

  • Capture exceções específicas, não genéricas
  • Nunca deixe blocos catch vazios
  • Use try-with-resources para recursos
  • Registre informações úteis sobre a exceção
  • Não abuse de "throws Exception"

Manipulação de Arquivos

  • Sempre feche recursos após o uso
  • Trate exceções de I/O adequadamente
  • Verifique existência e permissões dos arquivos
  • Considere o encoding dos arquivos de texto

Interface Gráfica Swing

  • Use o Event Dispatch Thread (EDT)
  • Separe a lógica da interface (MVC)
  • Evite operações demoradas na thread da GUI
  • Use layout managers em vez de posições absolutas
  • Forneça feedback visual para operações longas
  • Mantenha a interface intuitiva e responsiva
SwingWorker: Para operações demoradas como acesso a arquivos grandes ou operações de rede, use a classe SwingWorker para executar em segundo plano sem bloquear a interface.

Resumo

Tratamento de Exceções

  • Blocos try-catch para tratamento de erros
  • Checked vs. Unchecked exceptions
  • Try-with-resources para gerenciamento automático de recursos
  • Exceções personalizadas para representar erros de domínio

Gerenciamento de Arquivos

  • Classes java.io e java.nio para operações com arquivos
  • Leitura e escrita de texto e dados binários
  • Serialização para salvar objetos em disco

Interface Gráfica Swing

  • Componentes visuais para interação com o usuário
  • Layout managers para organização da interface
  • Tratamento de eventos para responder a ações do usuário
  • Diálogos para interações comuns
Integração: Juntos, esses elementos permitem criar aplicações robustas com interfaces gráficas que podem persistir dados e tratar de forma elegante situações de erro.
